## 🎯 Goal
Make every service repo's API usable by LLM agents as an installable **Agent Skill**
([open standard](https://agentskills.io)): standardized entry points wrapped by a `SKILL.md` on
a dedicated `agent-skill` branch per repo, with the frontend UIs and `service/README.md`s
documenting API usage.

## 📦 What exists now
- **Exemplar**: `atrium-page-classification@agent-skill` — root `SKILL.md`, zero-dep stdlib
  client `scripts/atrium_classify.py` (hand-rolled multipart, 3× retry on 502/503/504, exit
  codes 0/1/2/3), idempotent `scripts/server.sh`, flattened/trimmed tree,
  `small_data_samples/`. Four defects to fix (doc-vs-file `serve.sh`/`server.sh`, phantom
  paradata claim, phantom `frontend-lindat/` reference, response-schema over-documentation) —
  each generalized into a rule/CI check in the standard.
- **Service maturity gradient**: nlp-enrich (full meta-contract incl. `/health` + async jobs) >
  alto-postprocess (good docs, no `/health`, localhost CORS default) > translator (**no
  service README/frontend**, `MAX_UPLOAD_BYTES`, `400` misuse) > llm-enrich (**no HTTP API** —
  CLI only; gets a minimal FastAPI `service/` with `/extract_keywords`[`_text`] reusing
  `api_util/` parsers).
- **The standard** (doc §§4–9): uniform `GET /info` (`service`/`version`/`endpoints`/`limits`)
  + `GET /health` (+`?deep`), error table 413/415/422/429/500/503, canonical `MAX_UPLOAD_MB`,
  client env `ATRIUM_{PC,TR,AP,NE,LE}_URL`, domain endpoints untouched; no committed OpenAPI
  snapshots (live `/openapi.json` + contract tests instead); frontends get an API footer
  (curl + `/docs` + `/openapi.json` links).

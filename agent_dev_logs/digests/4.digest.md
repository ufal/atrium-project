## 🎯 Goal
Decide the translation base model — and, more durably, make the translation step **backend-swappable** so the model choice stays easy to extend (free + low-resource backends prioritized).

## 🧩 Decision — pluggable backends behind one seam
A `TranslationBackend` contract with a `get_backend(...)` resolution seam, implemented across Phases 0–3 on `test`. Three backends:
- **LINDAT** — default (`translation_backend = lindat`).
- **`openai_compatible`** — LLM backend (registered).
- **CTranslate2 (NMT)** — registration present (commented for now).

## 🔧 Hardening / remediation cycle (current `test` HEAD)
- 🧠 **No silent truncation:** configurable `max_tokens` for the LLM backend; `max_decoding_length` bumped to `2048` for the CT2 NMT path so long archival chunks don't terminate early.
- 🧠 **CT2 faithfulness guards:** replicated input/output length-ratio checks (`_MIN_LEN_RATIO` / `_MAX_LEN_RATIO`) to reject hallucinations / truncation.
- 🧠 **Precision glossary:** Tag-and-Protect glossary injection now matches on strict word boundaries (`\b`) so short keys don't over-match inside unrelated words.
- 🔌 **API streaming race fixed:** the FastAPI `/translate` endpoint was `FileResponse`-streaming the XML *after* the `TemporaryDirectory` block had deleted it; now buffers into an in-memory `Response`. Added `TestClient` coverage for the successful POST path.
- ⚡ **XSD validation** refactored/optimized; paradata/deployment compliance kept backend-aware.

## 🎯 Goal
Provide a Docker-compose + GitHub-Action wrapper for the four ARUP-CAS CU repo forks. Opened by motyc.

## 🧠 Strategy (settled)
Separate the **common ground** (a reusable workflow, `on: workflow_call` — the single source of truth) from the **per-repo differences** (thin caller workflows). Pilot/reference template: **atrium-translator**. Reuse what already exists rather than reinvent: `atrium_paradata.py` already reads `ATRIUM_RUNNER_IMAGE` / `ATRIUM_RUNNER_REPO` / `ATRIUM_RUNNER_REF`, so **GHCR-published, self-identifying containers** are the mechanism. motyc: orchestration to be discussed with rharasim; **no overall wrapper needed** — containers are reachable via API (this scopes the issue down to packaging + CI, not a runtime orchestrator).

## ✅ Landed
Commit `676a1fe` — centralized DRY CI/CD: `ci-cd-strategy.md`, `docker-tool.reusable.yml`, a caller example, shared `.coveragerc` / `ruff.toml`, and a Dependabot appendix. All four repos pass GH Actions; docs updated for rharasim to test.

## 🐞 Cross-cutting bugs (shared template — fixes apply to all four repos)
1. **Codecov upload never fires.** The "Upload to Codecov" step gates on `if: env.CODECOV_TOKEN != ''`, but `CODECOV_TOKEN` is never written to `env:` (only passed via `with: token:`). Condition is always `'' != ''` → false. Fix: `if: ${{ secrets.CODECOV_TOKEN != '' }}`.
2. **`@main` vs `@test` pin drift** between the templates' own example callers (`docker.caller.example.yml` → `@test`, `security.caller.example.yml` → `@main`). Standardize one ref convention before rollout.
3. **Version floor:** checkout@v5, setup-python@v6, codeql-action@v4, gitleaks-action@v3 (v2 EOL Sep 2026), trivy-action ≥0.36.0 SHA-pinned, Node24 baseline.

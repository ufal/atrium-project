- **As-built design diverges from the original plan**: `llm_run.py` stays a **local-only** entry point
  (`BACKEND ∈ {transformers, vllm}`); OpenRouter/Ollama are **separate entry points** sharing
  `llm_client_shared.py`, not a single 4-way `BACKEND` dispatcher — because `llm_utils.py` imports
  `torch` at module scope, and routing remote-only users through it would force the GPU stack on them.
  See the *As-built architecture* section in [`plans/24.plan.md`](../plans/24.plan.md).
- Post-bootstrap fixes: `Dockerfile` (was a broken `nlp-enrich` copy referencing dropped files),

## 🔧 2026-07-03 review pass (4 new modules only)
5 findings, all unit-tested (10 new tests → 47 total, green; ruff clean):
1. **[correctness]** `--attach-as-file` was dead code — `run_document_level()` never called
   `_build_attachment_content()`, so OpenRouter's file-attachment path (a specific #24 ask) did
   nothing. Wired via an optional `user_content_builder`.
2. **[correctness]** Ollama `ensure_model_pulled` over-matched tags (`qwen2.5:14b` satisfied a request
   for `qwen2.5:7b`) — now requires the exact tag; bare name only satisfied by `<name>:latest`.
3. **[simplification]** ~60 duplicated lines across `build_system_prompt`/`build_document_system_prompt`
   consolidated into shared helpers.
4. **[cleanup]** `OUTPUT_DIR` default inconsistency resolved across both clients.
5. **[robustness]** `run_line_level` no longer silently drops rows with blank/non-numeric
   `page_num`/`line_num`; coerced to `0` instead.

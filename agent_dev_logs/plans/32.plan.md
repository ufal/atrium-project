## Context
Standardize each service repo's HTTP API to the **OpenAPI-followed meta-contract** of
`docs/agent_skill_strategy.md` §4, then enforce it in CI so it can't drift — the same federation
model that `para-drift.reusable.yml` uses for the paradata trio. This plan is the issue-level index;
§4 of the strategy doc is the source of truth for the contract itself.

Decisions locked at planning time (unchanged):
1. **Meta-contract only** — uniform `GET /info` (`service`/`version`/`endpoints`/`limits`) +
   `GET /health` (shallow + `?deep=true`), §4.4 error table (413/415/422/429/500/502/503), canonical
   `MAX_UPLOAD_MB` + `ALLOWED_ORIGINS` default `*`, version from `para_config.txt`. **Domain endpoints
   not renamed** (`/predict_*`, `/translate`, `/process`, `/enrich`, `/extract_keywords`).
2. **Build llm-enrich a `service/` first** (it had none) — all five repos share one server standard.
3. **No committed OpenAPI snapshots** — validate the live `/openapi.json` (spec-validator) + the
   `/info`/`/health`/endpoint-set assertions via per-repo `tests/test_api_contract.py`; the hub
   `api-contract.reusable.yml` runs it, and `para-drift` guards `service/atrium_service.py` parity.
4. **Enforce-by-copy** — one canonical `docs/templates/shared/atrium_service.py`, byte-identical in
   each `service/` (not a monorepo, not submodules; §5.H model).

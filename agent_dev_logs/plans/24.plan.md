## As-built architecture (reconciles original plan item #5)
The original plan proposed a **single unified `BACKEND ‚àà {transformers, vllm, ollama, openrouter}`
dispatch inside `llm_run.py`**. The as-built design deliberately took a **different, cleaner shape**,
because `llm_utils.py` (imported by `llm_run.py`) unconditionally imports `torch` and sets
`PYTORCH_CUDA_ALLOC_CONF` on import ‚Äî so routing the remote/lightweight paths through `llm_run.py`
would force the GPU stack on remote-only users. Instead:

- **`llm_run.py`** stays the **local** entry point and accepts `BACKEND ‚àà {transformers, vllm}` only
  (raises on anything else) ‚Äî the heavy `torch`/vLLM path.
- **`openrouter_client.py`** and **`ollama_client.py`** are **separate entry points** that need only
  `requirements_remote.txt` (no torch), sharing all the pure-Python logic ‚Äî config loading, quality
  filter, context-window builder, archaeological schema + Pydantic validation ‚Äî via
  **`llm_client_shared.py`**, a front-end hand-synced with (not imported from) `llm_utils.py`.

`BACKEND` therefore selects the *local* engine; the remote/lightweight backends are selected by
running their own script. `llm_config.txt` documents both groups and their env knobs.

## What was ADDED (new work ‚Äî the point of #24)
1. **`openrouter_client.py`** ‚Äî remote LLM-as-a-service. `OPENROUTER_API_KEY`; provider routing incl.
   `--provider-data-collection deny` (data-sovereignty per #24); optional **file attachment** of
   `.md`/`.txt` documents (`--attach-as-file`). Reuses the shared Pydantic schema + validation.
2. **`ollama_client.py`** ‚Äî local Ollama server (`/api/chat`) with the native JSON `format` schema for
   structured output + auto model-pull. The lightweight local alternative to transformers/vLLM.
3. **`api_util/xml_to_md.py`** ‚Äî **TEITOK/ALTO ‚Üí Markdown/plain-text** converter, so whole documents
   can be fed to LLMs (document-level input) alongside the line-level CSV/TEITOK reader.
4. **`llm_client_shared.py`** ‚Äî the torch-free shared front-end (see *As-built architecture*).
5. **Multi-GPU from day one** ‚Äî `requirements_llm.txt` pins + `llm_config.txt` knobs (`BACKEND=vllm`,
   `TENSOR_PARALLEL_SIZE`, `CPU_OFFLOAD_GB`, `GPU_MEMORY_UTILIZATION`; #26/#27 recipes documented);
   `docker-compose.gpu.yaml` with GPU reservations.
6. **Tests + CI** ‚Äî `tests/` (LLM-only), 8 `.github/workflows/` callers + `dependabot.yml`, two
   `docker-compose*.yaml` profile-gated services, `CONTRIBUTING.md`.

## Refinements (2026-07-03 review pass ‚Äî the 4 new modules only)
A full correctness/cleanup review of the four genuinely-new modules; engine copies left untouched so
`nlp-enrich` parity holds. All fixes covered by unit tests (10 new; 47 total green).

1. **[correctness] OpenRouter `--attach-as-file` was dead code.** `_build_attachment_content()` and
   the flag were documented but never called ‚Äî `run_document_level()` always inlined the text, so the
   file-attachment path (a specific #24 ask) silently did nothing. Fixed by threading an optional
   `user_content_builder` through `run_document_level()`; the client now routes the document body
   through `_build_attachment_content(..., args.attach_as_file)`.
2. **[correctness] Ollama `ensure_model_pulled` over-matched tags.** The bare-name match treated
   `qwen2.5:14b` as satisfying a request for `qwen2.5:7b`, skipping the pull and then running against
   a tag Ollama didn't have (and could crash on a `None` tag name). Fixed to require the exact tag,
   with a bare name also satisfied by `<name>:latest` only.
3. **[simplification] De-duplicated ~60 lines** shared by `build_system_prompt` /
   `build_document_system_prompt` (vocab collection + prompt render + binary-search truncation) into
   `_collect_vocab_terms` / `_render_vocab_prompt` / `_fit_vocab_prompt`, halving the hand-sync burden
   the module's own docstring warns about. Behaviour preserved.
4. **[cleanup] `OUTPUT_DIR` default inconsistency** in both clients (two different literal defaults
   for the same key) resolved into a single base path.
5. **[robustness] `run_line_level` silent row drop** ‚Äî a blank/non-numeric `page_num`/`line_num` was
   mislabelled as a filter-skip and dropped; now coerced to `0` so the line is still processed.

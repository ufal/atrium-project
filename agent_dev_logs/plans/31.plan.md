# Strategy — Issue #31: AGENT SKILL — based on API service to each repo

> Issue [#31](https://github.com/ufal/atrium-project/issues/31) · Labels: `enhancement`, `development` · Milestone: Q3 [WP8/WP7] · **Status: 🟢 Implemented on all five `agent-skill` branches (2026-07-18) → refinement stage planned (validation · acceptance · consistency · release)**

## Context
Wrap each service repo's API in an **Agent Skill** (`SKILL.md` per the
[open standard](https://agentskills.io) — the cross-vendor format adopted by Claude Code, Codex,
Copilot, Antigravity, Gemini CLI, …), on a dedicated **`agent-skill` branch** per repo. The full
normative strategy — service meta-contract, branch anatomy, client contract, SKILL.md template,
per-repo work plans — is **[`docs/agentskillstrategy.md`](../../docs/agentskillstrategy.md)**; this
plan is the issue-level index of it.

Decisions locked at planning time (unchanged):
1. **llm-enrich got a minimal FastAPI `service/` first** (mirrors nlp-enrich) — all five repos share
   one server–client standard.
2. **Meta-contract standardization only**: uniform `GET /info` + `GET /health`, multipart
   conventions, error-code table (413/415/422/429/500/503), env vars (`ALLOWED_ORIGINS`, canonical
   `MAX_UPLOAD_MB`, client `ATRIUM_<XX>_URL`), version from `para_config.txt`. Domain endpoints
   (`/predict_*`, `/translate`, `/process`, `/enrich`, `/extract_keywords`) **not** renamed.
3. **No committed OpenAPI snapshots** — agents fetch the live `/openapi.json`; drift caught by
   per-repo contract tests + the hub `skill-validate.reusable.yml`.

Refinement-stage decisions (2026-07-18):
4. **All workstreams W1–W5 in scope.**
5. **Drift model = manual for now** — a documented sync checklist, no automated forward-merge or
   regenerate-from-default. (Push-triggered skill-validate CI still catches breakage; it just
   doesn't auto-sync.)

## Strategic actions (fix-before-rollout)

### A — Shared-template bugs (apply once, propagate to all four)
1. **Codecov never uploads.** The step gates on `env.CODECOV_TOKEN` which is never written to
   `env:`. Fix: `if: ${{ secrets.CODECOV_TOKEN != '' }}`.
2. **Ref-pin drift.** `docker.caller.example.yml` pins `@test`, `security.caller.example.yml`
   pins `@main`. Choose one convention (the ecosystem authoritative branch is `test`) and make
   every caller agree before rollout.
3. **Action version floor.** checkout@v5, setup-python@v6, codeql-action@v4, gitleaks-action@v3
   (v2 EOL **Sep 2026**), trivy-action ≥0.36.0 SHA-pinned, Node24 baseline.

### B — translator reference-caller fixes (it is the pilot template)
| Pri    | Fix                                                                                                                                                         |
|--------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **P0** | Triggers `branches:[main]` → `branches:[test, master]` (real branches; workflows currently never fire on real activity).                                    |
| **P0** | `security.yml` passes `image-name`; the reusable only declares `image-ref` → call fails validation. Use `image-ref: ghcr.io/ufal/atrium-translator:latest`. |
| **P1** | `docker.yml` passes `pre-install-torch: true` (bool) into a `type: string` input → `pip install true`. Quote it.                                            |

## 🛠️ Update Plan: Cross-Cutting Shared Template Bugs

Before a full-scale rollout, several bugs within the core `docker-tool.reusable.yml` template must be resolved:

* **Codecov Upload Fix:** The "Upload to Codecov" step currently never fires because it gates on 
`if: env.CODECOV_TOKEN != ''`, but the token is never written to `env:`. The condition must be fixed to 
`if: ${{ secrets.CODECOV_TOKEN != '' }}`.

* **Reference Pin Standardization:** There is a tracking drift between the example templates; some reference `@test`
while others reference `@main`. Workflows will be standardized to follow the authoritative ecosystem branch (`test`).

* **Action Version Floor Upgrade:** Ensure all underlying major workflow actions are modernized to a Node 24 baseline
(including `checkout@v5`, `setup-python@v6`, `codeql-action@v4`, and `gitleaks-action@v3` to prevent the upcoming 
September 2026 v2 EOL).

### 1. atrium-translator (Pilot Template Reference)

* **P0 Trigger Fix:** Update workflows from `branches: [main]` to `branches: [test, master]` so they actually fire on

* **P1 Torch Dependency Cleanup:** Remove the accidental `pre-install-torch: true` string-coercion parameter since the 
translator has zero PyTorch requirements.


### 2. atrium-page-classification

is ready.

* **P1 Dependency Realignment:** Align and verify the current discrepancy between `setup/requirements.txt` and 
`service/requirements.txt` regarding PyTorch versions.

  
### 3. atrium-nlp-enrich

* **P1 Upstream Stability Monitoring:** Deploy scheduled smoke tests to aggressively check the LINDAT API endpoint paths,
which represent the highest point of upstream fragility across the project.

* **P1 CodeQL Analysis:** Implement CodeQL scanning across its FastAPI endpoints to monitor subprocess-spawning command
injection risks.



### 4. atrium-alto-postprocess

* **P1 Heavyweight Vulnerability Audits:** Implement comprehensive `Trivy` container scanning due to the heavy 
combination of PyTorch, LayoutReader, and fastText binaries.

* **P1 Reachability Checks:** Set up automated network health monitors checking external dependencies like
`FreeOCR-AI/layoutreader` and Hugging Face model pull stability.

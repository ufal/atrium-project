### Phase 1 — Close Thread G stragglers

### Phase 4 — Release & policy hygiene

## Tracking aids
- Canonical tracker: `docs/plan_repo_review.md`. Evidence base: `digests/issue10_alignment_1307.md` (claim-by-claim `file:line` verdicts).
- Tier-2 iterations: fresh venv per repo from that repo's own `requirements*-test.txt` — the shared-venv shortcut masks under-declared deps (how alto's gap stayed hidden).

## Risks
- **Unmanaged duplication (nlp↔llm)**: the exact cross-repo drift failure mode this plan previously flagged for `para_licenses.py` is now reproducing in the LLM modules — Phase 2 is the mitigation.
- **Doc staleness**: execution now outpaces the documents; stale "FAILING" claims (e.g. Shellcheck) mislead later review rounds. Reconcile docs at the end of every round.
- **Endless refactoring loops**: LLMs might suggest stylistic rewrites; strictly limit changes to architectural fixes, tests, and bug resolution.

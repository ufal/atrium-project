## Strategic actions (Phased execution)

### Phase 0 — Release hygiene (Immediate, small)

1. **Re-tag nlp**: delete/re-publish `v1.16.2` as `v0.16.2` so the tag matches `CITATION.cff`/`para_config.txt`. (Alto's malformed `v1.0.0.-beta` needs no action — superseded by the clean `v1.1.0-beta`.)
2. **Harden the gate**: make `security.reusable.yml` fail the release job on any tag ≠ CITATION ≠ `para_config` exact-match (after `v`-prefix normalization only). This is the one fix that prevents the whole class from recurring; individual re-tags don't.

### Phase 1 — Execute the 2026-07-16 round (in comment order)

3. **alto → llm → nlp → translator**: per-repo Tier 1 + Tier 2 in a clean venv from that repo's own `requirements*-test.txt` (first real exercise of the new self-contained files — record fresh pass/skip/coverage baselines while at it).
4. **hub (`atrium-project`) — first-ever per-repo pass**: define Tier-1 scope for a non-Python-package repo: workflow YAML lint, reusable-workflow input/secret contracts, template↔consumer sync (`docs/templates/*` vs the 5 repos), docs link hygiene (the dangling `issue10_alignment_1307.md` citations are a first finding), e2e smoke scripts compile.

### Phase 2 — Shared-code governance for the LLM stack

5. **Decide the canonical home** for `llm_utils.py` / `vocab_manager.py` / `llm_run.py`: either extend the `para-drift` check to cover them, or declare `llm-enrich` canonical and prune nlp's copies. Urgency has **grown** since 07-13: llm v0.2.0/v0.3.0 added DU code, `xml_to_md`, and remote clients on top of the already-diverged `llm_run.py`.

## Risks

- **Release plumbing over code**: the gate that should catch tag/version mismatch demonstrably doesn't — until Phase 0.2 lands, every auto-release is a chance for a new mistag.
- **Unmanaged duplication (nlp↔llm)**: the exact cross-repo drift failure mode this plan previously flagged for `para_licenses.py` is reproducing in the LLM modules, and the delta grows with every llm release — Phase 2 is the mitigation.
- **Doc staleness / cadence mismatch**: execution outpaces the documents within hours, not weeks, on multi-release days; stale claims (alto tag, llm date, "Thread G open") mislead later rounds. Reconcile docs at the end of every round and follow the Phase-4 cadence guard.
- **Endless refactoring loops**: LLMs might suggest stylistic rewrites; strictly limit changes to architectural fixes, tests, and bug resolution.

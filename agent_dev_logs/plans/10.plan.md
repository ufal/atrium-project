## Strategic actions (Phased execution)

### Phase 0 — Release integrity — ✅ DONE
Re-tag (nlp `v0.16.2`) and gate (inline `version-guard` in every `release.yml`) both landed. The
mistag/malformed-tag class can no longer reach publication. A DRY `release-guard.reusable.yml`
remains an optional future consolidation once the inline guard has proven itself in the wild.

### Phase 1 — Execute the 2026-07-16 round — ✅ MOSTLY DONE
- **alto → llm → nlp → translator**: the per-repo code-quality pass landed (correctness + tests +
  hygiene; §P1c/§P2t in `plan_repo_review.md`); fast lanes re-run green above the `fail_under` floors.
- **hub (`atrium-project`)**: first-ever pass, partial — the stale tracker was refreshed and the
  dangling `issue10_alignment_1307.md` citation dropped. Still to do: a fuller CI/template sweep
  (reusable input/secret contracts, `docs/templates/*` ↔ consumer sync, e2e-smoke compile check).

### Phase 2 — nlp↔llm shared-engine governance — ⏸️ DEFERRED
Decision held by the maintainer (2026-07-22). Measured divergence at the current HEADs:
`vocab_manager.py` byte-identical; `llm_run.py` differs by one line (`program=` string);
`llm_utils.py` forked (+343 lines in llm — the auto CPU-offload subsystem). **Do not** naively add
these to para-drift (they diverge; CI would redden on day one). When revisited, the recommended
direction is to declare **llm-enrich canonical**, reconcile or explicitly fork nlp's copies, and
only then extend the drift check.

### Phase 3 — Heavy refactors (need a GPU-capable env) — ⛔ NOT DONE HERE
- **Modularization** of the >800-line modules — alto `text_util_langID.py` (1560 L) into
  scoring/categorize/prefilter, `langID_classify.py` (938 L) extracting `build_line_record()` (also
  the Phase-0 blocker of the #23 quality-model plan) + `apply_document_postprocessing`; pc `run.py`
  (1661 L) into a `commands/` package. **The safety gate is byte-identical output CSVs on the
  `data_samples/` fixtures**, which requires the GPU Qwen-0.5B perplexity worker — unavailable in a
  CPU-only session, so a blind refactor is not safe. Do this on a GPU branch and coordinate the alto
  `build_line_record()` extraction with issue #23 so it happens once.
- **print→logging** — deprioritised: the plan's own guardrail is "strictly limit changes to
  architectural fixes, tests, and bug resolution"; this is largely stylistic and changes stdout→stderr
  routing where the alto CPU/GPU workers deliberately `print(..., flush=True)`.

### Phase 4 — Policy & docs hygiene — ⏳ PARTIAL
1. **gitleaks**: adopt `secret-scan.caller.example.yml` per repo once ARUB/ARUP sign-off + license land.
2. **translator doc move**: `git mv translation_backends.md docs/translation-backends.md` (all code refs
   already point at the `docs/` path).
3. **Forward-merge `test` → `main`** so default-branch readers see `project_state_2207.md`, the
   refreshed digest/plan, and `plan_repo_review.md`.
4. **Ratchet-up**: raise each repo's `fail_under` toward its measured fast-lane coverage.
5. **Cadence guard**: re-check `list_releases` + CITATION at digest-write time; timestamp release tables.

## Risks
- **Unmanaged duplication (nlp↔llm)**: the fork widens with every llm release; deferred, not closed.
- **Refactors without a runtime**: modularizing the scoring engine without the GPU parity check risks
  silent behaviour change — gate on the fixtures, don't eyeball it.
- **Doc staleness / cadence mismatch**: execution outpaces the documents within hours on multi-commit
  days; reconcile docs at the end of every round.
- **Endless refactoring loops**: keep changes to architectural fixes, tests, and bug resolution — not
  stylistic sweeps.

## Strategic actions (Phased execution)

### Phase 0 — Unblock & Sync (Immediate)
1. **Fix `nlp` Shellcheck**: Resolve the active CI failure.
2. **Version Sync**: Update `service/api.py` in `pc` and `translator` to match their actual CITATION/release versions.
3. **Secret Scan Check**: Verify `security.reusable.yml` properly executes `gitleaks`.

### Phase 1 — Test Isolation (Thread G)
4. **Dependency Hygiene**: Ensure every repo has a functional, self-contained `requirements-test.txt` so Tier-2 `pytest` collection doesn't crash on heavy deps (torch, fasttext).

### Phase 2 — Test the Untested
5. **Cross-repo `para_licenses.py`**: Port the deduplication logic from `alto` to all other repos and write the missing shared test suite.
6. **Coverage Gaps**: Target 0% coverage spots identified by Opus (e.g., translator `load_vocab`, subprocess-only tests in alto/pc).

### Phase 3 — Strict Linting & Release
7. **Ruff Alignment**: Adjust `nlp` ruff config to match the advisory-first policy (remove `--exit-non-zero-on-fix` if inappropriately blocking).
8. **Finalize**: Ensure `date-released` fields are fresh and paradata-provenance tracking is rock solid.

## Tracking aids
- All execution must be strictly mapped against `docs/plan_repo_review.md`. Use the `test` branch shared venv (`/tmp/atrium-venv`) for rapid Tier-2 iterations without bloating local environments.

## Risks
- **Cross-repo drift**: Fixing an issue in `alto` without propagating it to `pc`/`nlp`/`translator` (as happened with `para_licenses.py`).
- **Endless refactoring loops**: LLMs might suggest stylistic rewrites; strictly limit changes to architectural fixes, tests, and bug resolution.

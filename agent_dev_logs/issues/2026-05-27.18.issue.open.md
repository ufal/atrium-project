The repos are already **pre-wired for this**: the shared `atrium_paradata.py` provenance logger reads `ATRIUM_RUNNER_IMAGE`, `ATRIUM_RUNNER_REPO`, `ATRIUM_RUNNER_REF` env vars and records them in every output JSON, with the documented example `ghcr.io/ufal/atrium-translator:v0.5.2`. The maintainers clearly intended GHCR-published, self-identifying containers â€” this plan delivers exactly that.

**Outcome:** anyone can `docker compose run <tool>` over a mounted data dir with zero manual setup; tagged releases auto-publish a provenance-stamped image to GHCR; PRs are gated by the existing fast test suite.

#### Decisions (confirmed with user)

- **Scope:** the **4 component repos only**. `atrium-project` gets just a short strategy/index doc â€” no umbrella orchestration compose.
- **GH Action** (no preference given â†’ recommended): **build & publish to GHCR** on tag/release, feeding the paradata `ATRIUM_RUNNER_*` hook; **`pytest -m "not slow"`** gate on PRs.
- **Run mode:** batch CLI (one-shot, mounted inâ†’out, then exit) by default; an optional `api` Compose profile only for the two repos that ship a FastAPI `service/` (alto-postprocess, page-classification). translator & nlp-enrich are batch-only (confirmed: no `service/` dir).
- **Rollout:** pilot **atrium-translator** (lightest â€” no GPU, no API), verify end-to-end, then replicate the proven template to the other three.
- **Branch:** all work on `claude/determined-hypatia-fi57y4` (already exists in every repo; remote `origin` â†’ `ufal/<repo>`).

#### Reuse â€” existing things to wire into, not reinvent

- **Provenance contract:** `atrium_paradata.py` (every repo) reads `ATRIUM_RUNNER_IMAGE/REPO/REF` (translator lines 54-58, 144-148, 253-255). Dockerfile build-ARGs â†’ ENV must set these; nothing in app code changes.
- **Version source:** `para_config.txt` `[tool] version` (e.g. page-classification `v1.1.1-beta`) â€” drives image tags.
- **Test gate:** existing `pytest.ini` with the `slow` marker in all repos â†’ CI runs `pytest -m "not slow"` (network/GPU-free).
- **Setup logic to translate into Dockerfile steps (not re-run at runtime):** `atrium-alto-postprocess/setup_api_server.sh` (LayoutReader `v3/` sparse-checkout + `lid.176.bin` download), `atrium-page-classification/setup_api_service.sh`.
- **Existing `.gitignore`s** (present in 3 of 4; absent in alto-postprocess) inform the `.dockerignore`.

#### The templated pattern (identical vs parameterized)

Every repo gets the same 4 files: `Dockerfile`, `docker-compose.yml`, `.dockerignore`, `.github/workflows/docker.yml`. The two service repos also get `docker-compose.gpu.yml` and an `api` profile.

**Identical across all four:** the `ARG ATRIUM_RUNNER_* â†’ ENV` block; `PYTHONUNBUFFERED`/
`PYTHONDONTWRITEBYTECODE`/pip env; `HF_HOME=/cache/huggingface`; non-root user `atrium` (uid 10001) owning `/app /cache /data`; layer order (copy requirements â†’ `pip install` â†’ copy source); volume convention (`./data:/data` bind + `hf-cache` named volume); the whole CI workflow except `IMAGE_NAME` and the test-deps apt line; the `.dockerignore`.

#### Pilot: atrium-translator (reference template â€” full contents)

##### `Dockerfile`

```dockerfile
# syntax=docker/dockerfile:1.7
FROM python:3.11-slim AS base

ARG ATRIUM_RUNNER_IMAGE=""
ARG ATRIUM_RUNNER_REPO="https://github.com/ufal/atrium-translator"
ARG ATRIUM_RUNNER_REF=""
ENV ATRIUM_RUNNER_IMAGE=${ATRIUM_RUNNER_IMAGE} \
    ATRIUM_RUNNER_REPO=${ATRIUM_RUNNER_REPO} \
    ATRIUM_RUNNER_REF=${ATRIUM_RUNNER_REF} \
    PYTHONUNBUFFERED=1 PYTHONDONTWRITEBYTECODE=1 \
    PIP_NO_CACHE_DIR=1 PIP_DISABLE_PIP_VERSION_CHECK=1 \
    HF_HOME=/cache/huggingface

RUN apt-get update && apt-get install -y --no-install-recommends \
        build-essential g++ libxml2-dev libxslt1-dev ca-certificates \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt requirements-test.txt ./
RUN pip install -r requirements.txt -r requirements-test.txt
COPY . .

RUN useradd --create-home --uid 10001 atrium \
    && mkdir -p /cache/huggingface /data \
    && chown -R atrium:atrium /app /cache /data
USER atrium

ENTRYPOINT ["python", "main.py"]
CMD ["/data/input", "--alto", "--formats", "alto.xml", "--target_lang", "en", "-o", "/data/output"]
```

#### Shared conventions

- **HF cache:** `HF_HOME=/cache/huggingface` + named volume `hf-cache` â€” models never baked, downloaded once, reused.
- **Model dirs:** `alto-models:/app/models` (holds `lid.176.bin`), `page-model:/app/model`.
- **I/O:** single bind `./data:/data`; convention `input/` + `output/` subdirs; paradata JSON lands under the output dir's `paradata/`.
- **Secrets:** `HF_TOKEN` supplied at runtime only (compose `environment:` from a git-ignored `.env`, or compose secrets) â€” **never** a build ARG, never committed.
- **`.env`** (git-ignored): `ATRIUM_VERSION`, `ATRIUM_RUNNER_REF`, `HF_TOKEN`.
- **GPU (alto, page, nlp-llm):** default images are CPU. Ship `docker-compose.gpu.yml` adding `deploy.resources.reservations.devices` (driver `nvidia`, `capabilities:[gpu]`) + `NVIDIA_VISIBLE_DEVICES`; used via `-f docker-compose.yml -f docker-compose.gpu.yml`. Requires NVIDIA Container Toolkit on host. We keep the slim base + torch's bundled CUDA rather than an `nvidia/cuda` base (smaller, no CUDA-version coupling).
- **atrium-project:** add `docs/containerization.md` (this strategy, condensed) + a links table; no code.

### Shared foundation (identical by design)

- **Base & build:** `python:3.11-slim`, BuildKit (`# syntax=docker/dockerfile:1.7`), a single
  `base` stage (nlp-enrich adds a second â€” see below).
- **Provenance contract:** `ARG â†’ ENV ATRIUM_RUNNER_IMAGE / ATRIUM_RUNNER_REPO / ATRIUM_RUNNER_REF`,
  consumed by the shared `atrium_paradata.py` and stamped into every output paradata JSON as
  `docker_image` / `repository` / `runner_ref`. CI injects them as build-args
  (`github.repository`, `github.sha`, `IMAGE:ref_name`), so a published image self-reports its origin.
- **Runtime env:** `PYTHONUNBUFFERED`, `PYTHONDONTWRITEBYTECODE`, `PIP_NO_CACHE_DIR`,

- **Layer order:** copy requirements â†’ `pip install` â†’ `COPY . .` (cache-friendly).
- **Non-root user** `atrium` (uid 10001) owning `/app`, `/cache`, `/data`.
- **Compose:** `image: ghcr.io/ufal/atrium-<tool>:${ATRIUM_VERSION:-dev}`, build passes the
  provenance args, `environment:` mirrors them + `HF_HOME`, volumes = `./data:/data` + a named
  `hf-cache` volume + a read-only bind of the tool's config file. HF models are **never baked** â€”
  pulled at runtime into `hf-cache`.
- **CI (`.github/workflows/docker.yml`):** a `test` job runs `pytest -m "not slow"` on every
  PR/push; a `build-and-push` job runs **only on `v*` tags / releases** and pushes
  `ghcr.io/ufal/atrium-<tool>` tagged `{semver, sha, latest}` via `docker/metadata-action` +
  `docker/build-push-action` (GHA cache), with provenance passed as build-args.

**Cross-cutting notes**

- **GPU overlays:** alto, nlp, and page each ship a `docker-compose.gpu.*` adding NVIDIA device
  reservations; translator has none (CPU-only tool). page-classification is the only image that
  *requires* a GPU; the others are CPU-first with optional GPU.
- **API services:** alto and page expose an opt-in FastAPI on `:8000` behind an `api` profile;
  translator and nlp are batch-only (nlp instead has an `llm` profile for semantic enrichment).

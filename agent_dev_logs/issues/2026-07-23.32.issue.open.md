### Reference implementation to copy
`atrium-nlp-enrich/service/api.py` — already nails `/info` `service`+`limits` and `/health`
(shallow + `?deep=true`). Every other service is harmonized *toward* it. Version reader
pattern is the CONTRIBUTING.md canonical `_read_tool_version()` (dedupe the 4 copies).

### Work plan

#### 1. Shared helper `service/atrium_service.py` (new, one canonical copy)
Factor the repeated meta-contract logic so services stop copy-pasting it. Provide:
- `read_tool_version(service_dir)` — canonical reader (handles `para_config.txt` **and**
  `setup/para_config.txt`; strips leading `v`).
- `build_info(app, service, limits, **capabilities)` — returns `{service, version:
  app.version, endpoints:[r.path for API routes], limits, **capabilities}`.
- `attach_health(app, deep_check=None)` — registers shallow+deep `GET /health` (200 / 503).
- `resolve_max_upload_mb(default)` — reads `MAX_UPLOAD_MB`, deprecated `MAX_UPLOAD_BYTES`
  fallback for one release.
- `add_cors(app)` — standard CORS, `ALLOWED_ORIGINS` default `*`.

Place canonical copy in hub `docs/templates/shared/atrium_service.py`; copy byte-identical
into each repo's `service/`. (Consistent with the existing "not-a-monorepo, enforce-by-copy"
model — §5.H / §9.)

#### 2. Per-repo alignment (edit `api.py` to call the helper)
- **page-classification** `service/api.py`: `/info` → add `service`, `endpoints`, `limits`
  (`max_upload_mb`, `max_pdf_pages`); keep `categories`/`available_models` as capabilities.
  Add `/health`. `MAX_UPLOAD_BYTES` const → `resolve_max_upload_mb(10)`.
- **translator** `service/api.py`: `/info` `name`→`service`, add `endpoints`+`limits`; keep
  `supported_formats`. Add `/health` (deep: LINDAT reachable / backend key). `/translate`
  non-XML **400→422** (`~line 98`). Introduce `MAX_UPLOAD_MB` (keep `_BYTES` fallback).
- **alto-postprocess** `service/text_api.py`: `/info` add `service`+`endpoints`+`limits`
  (keep `status`/`device`/`supported_formats`/`quality_categories`/`line_fields`). Add
  `/health`. **Fix wrong 500s** for missing filename/content-type (`lines 144-147`) →
  400/415. Add `MAX_UPLOAD_MB` size guard (currently none) → 413. `ALLOWED_ORIGINS`
  default `*`; CORS methods align.
- **nlp-enrich** `service/api.py` (reference): smallest delta — add `endpoints` to `/info`;
  optionally adopt the shared helper for parity. `/health` already present.

#### 3. Build llm-enrich service (new `service/`)
New `service/api.py` + `service/__init__.py` + `service/requirements.txt` +
`service/README.md`, structured like nlp-enrich:
- `/info` (`service="atrium-llm-enrich"`, version from `para_config.txt` `v0.3.0`,
  `endpoints`, `limits`, capabilities: backends/models/langs).
- `/health` (deep: `OPENROUTER_API_KEY`/`OLLAMA_HOST` present + reachable).
- `POST /extract_keywords` (multipart `file`: TXT/CSV/TEITOK/ALTO) and
  `POST /extract_keywords_text` (JSON `{lines:[...]}`) → per-doc `extracted_keywords_cs/en`,
  wrapping `process_document` / `process_document_vllm` (from `llm_utils`, driven today by
  `llm_run.main`). Reuse `resolve_max_upload_mb`, error codes per §4.4.
- Dockerfile/docker-compose `api` profile on port 8000 (new stage mirroring nlp-enrich).

#### 4. Contract tests (§12.3) — `tests/test_api_contract.py` per repo
Using FastAPI `TestClient` + in-process `app.openapi()`, assert: `/info` `service`==repo-name,
non-empty `version`, `endpoints` matches the real route set and contains the §4.2 primary
paths, `limits.max_upload_mb` present; `/health` → 200 with `status` ∈ {ok,degraded};
`app.openapi()` validates via `openapi-spec-validator` (add to `requirements-test.txt`).

#### 5. Hub CI enforcement (deliverable files; not executed here since no push)
- New `atrium-project/.github/workflows/api-contract.reusable.yml` (sibling of
  `para-drift.reusable.yml`): runs each repo's contract test / boots app and validates
  `/openapi.json`. ~12-line caller `api-contract.yml` per repo.
- Extend `para-drift.reusable.yml` to also cover `service/atrium_service.py`.

#### 6. Docs
- `docs/plan_repo_review.md`: add workstream **§5.A — API meta-contract / OpenAPI
  conformance (#32)** + status-matrix rows (A1 `/info` envelope, A2 `/health`, A3 error-code
  harmonization, A4 contract test/CI, A5 llm-enrich service).
- Update the 4 service `README.md`s to the `serviceREADME.template.md` shape (new `/health`
  row, `/info` envelope). Mark §4 drift notes "landed" in `agent_skill_strategy.md`.

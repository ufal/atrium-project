## Post-review-round validation of the 4 ATRIUM tool repos

### Validation matrix (per repo)
Repos: `atrium-translator`, `atrium-page-classification`, `atrium-alto-postprocess`, `atrium-nlp-enrich`
(run in that order — lightest deps first).

- [x] **Tier 1 — static, no network (guaranteed):** 
- `python -m compileall -q .` — syntax/compile check of every module
- `ruff check --config <project-template ruff.toml> .` — mirrors the CI non-blocking lint; report rule-hit counts

- [x] **Tier 2 — fast tests, needs PyPI (best-effort):** 
- Create one throwaway venv (`/tmp/atrium-venv`); per repo `pip install -r requirements.txt [setup/requirements.txt] -r requirements-test.txt`
- `python -m pytest -m "not slow" --cov=. --cov-report=term-missing` (CI-equivalent test job)
- Record per repo: install result, collection result, passed / failed / skipped, coverage %

#### **Out of scope (reported with reason, not run):**
- Docker image builds / `compose up` (heavy, time) — note docker is available if the user wants a follow-up smoke build
- `@slow` tests (GPU/models/network), and nlp-enrich `requirements_llm.txt` / vLLM / GLM / torch-GPU paths

- [ ] Done


### Failure handling
- If PyPI is unreachable or a heavy dep (e.g. torch) won't install: clearly report it, still deliver Tier-1 results and any partial pytest collection rather than aborting the whole run.
- Capture exit codes; show short tails of real failures (compile errors, ruff findings, test failures) — not full logs.

- [x] Done

### Deliverable
- A consolidated **per-repo validation table** in chat: `compileall` | `ruff` (findings) | `pytest` (pass/fail/skip) | `coverage %` | notes.
- Call out any genuine regressions vs. expected/known gaps from the review plans.
- Nothing written into the repos; nothing pushed.

- [x] Done

## Approach — repeat per repo, in order pc → alto → nlp → translator
1. **Sync** the local clone to latest `test`: `git fetch origin test && git checkout -B test origin/test`
   (no local tracked changes exist — only ignored `__pycache__`/`.coverage` artifacts).
2. **Baseline the current state** with the shared venv `/tmp/atrium-venv` (deps already installed: torch, transformers,
   fastapi, httpx, pydantic, ultralytics, …): `python -m compileall -q .`, `ruff check --config
   /home/user/atrium-project/docs/templates/ruff.toml .`, `python -m pytest -m "not slow" --cov=. --cov-report=term-missing`.
3. **Re-map changed code** (targeted reads / one Explore agent) focusing on what changed vs the original plan baseline.
4. **Reconcile the committed plan's backlog**: for each P0/P1/P2 item mark **ADDRESSED / PARTIAL / OPEN** against the
   actual latest code, with `file:line` evidence.
5. **Review new/changed code** for fresh issues across the six aspects (architecture; merged pipeline & API service;
   unit-test coverage per function; Docker + GH Actions; file tree; CONTRIBUTING.md / release history).
6. **Produce concrete patch diffs** for actionable, low-risk fixes — **verified locally** against the `test` checkout
   (compile/ruff/pytest) before presenting.
7. **Deliver that repo's completed review** in chat (status table + new findings + verified patches), then move on.
